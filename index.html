MarioDupe

by: James Qian

This game was made for my level 2 project at The League of Amazing Programmers.
Click here to download and play!

1. Why I Made This Game

I made this game because I wanted to recreate a Mario game where they jump on platforms. This was my idea because Mario was the first thing that came to my mind when I thought of something like this.

2. What I Learned

Something that I learned from user testing is that I should add some images instead of blocks. Another feedback that I got is to make a "time alive" variable to count by seconds how long you were alive for in the game.

3. Architecture of My Game

The player and the platform class are all under the parent class called ObjectManager. Then everything gets called and gets coded in the GamePanel class. The GamePanel class is the main hub for all the code and the functions of it. The runner class runs the game for everyone to enjoy.

4. Technical Implementation

Something I am proud of is getting the platforms to work because that took a lot of class time to figure out. Another thing that I like about my game is the graphics and pictures that I found online, I think that they go really good together. Some steps that I took to accomplish the goal for platforms is I divided the things into parts and then I thought of how to do it.

5. Most Challenging Part

The most challenging part of this project was probably figuring out how to make the platforms spawn at a certain time. I figured out how to do it by implementing a timer. Another difficulty that I faced was how to make the character jump. I overcame this difficulty by looking at the demo games, and seeing how they implemented velocity and gravity into the game.

6. Future Fixes and Updates

Some improvements I would like to add to this game is to make bullets or something else that would make it harder to stay alive. I would also want to add different levels to the game and not just make it one boring thing, but add different game modes.
